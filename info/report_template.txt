1. INTRODUCTION

    Motivation: retrieval over multimodal documents (text + images in PDFs) is
    challenging because traditional pipelines are text-centric and miss visual signals.

    Research questions:
        - Does multimodal retrieval with Jina CLIP v2 image embeddings outperform
          text-only retrieval?
        - Which query technique maximises Precision@5?
        - Does the agentic pipeline outperform the non-agentic advanced system?

    Brief overview of the three systems:
        System 1 — Baseline RAG (text-only)
        System 2 — Advanced mRAG (multimodal, multiple query techniques)
        System 3 — Agentic mRAG (LLM-controlled agent pipeline)

    Paper structure summary.

6. SYSTEM 3 — AGENTIC mRAG

    Framework: LangChain / LlamaIndex

    Agent architecture (3 agents minimum):

        Agent 1 — Query Rewriting Agent
            Rewrites and expands the input query using LLM-based reasoning.
            Decision: determine whether the original query needs reformulation.

        Agent 2 — Retrieval Agent
            Decides the retrieval strategy and executes search over Qdrant.
            Decision: choose between text-only and multimodal retrieval based
            on query type.

        Agent 3 — Answer Generation Agent
            Synthesises retrieved evidence into a final answer.
            Decision: assess whether retrieved context is sufficient or
            additional retrieval is needed before generating.

    Each agent uses LLM-based decision-making, not just modular function calls.

    Generation model: qwen3-vl:8b on UiS Ollama cluster

7. EVALUATION FRAMEWORK

    Retrieval metrics (evaluated on retrieved document sets):
        - Precision@k  for k = 1, 3, 5
        - Recall@k     for k = 1, 3, 5
        - Page Recall@k (binary hit on correct PDF + overlapping page)
        - MAP — Mean Average Precision
        - MRR — Mean Reciprocal Rank

    Generation metrics (evaluated on generated answers):
        - Exact Match (after normalisation: lowercase, punctuation removal,
          number normalisation e.g. "1,358,000" → "1358000")
        - Token F1 — SQuAD-style token overlap using Counter intersection
        - Contains Match — ground truth appears anywhere in the prediction

    Supplementary metrics:
        - BLEU (note: designed for longer texts, limited meaning on short answers)
        - ROUGE-1, ROUGE-2, ROUGE-L
        - Semantic similarity (sentence-transformers cosine)

    Evaluation subset:
        First 20 pure-text questions from the train split.
        Justification: balances evaluation depth with runtime constraints.
        Limitation acknowledged in Section 10.

    Key findings (examples):
        - RAG-Fusion improved P@5 by X% over standard retrieval
        - HyDE achieved best R@5 among all query techniques
        - Multimodal retrieval improved results on image-based questions
          but had marginal effect on pure-text questions
        - Chain-of-thought prompting improved F1 for complex multi-step questions

    Limitations:
        - Evaluation subset limited to 20 questions due to runtime constraints
        - No fine-tuning performed on domain data
        - Multimodal retrieval constrained to available pre-indexed image collections
        - Agentic system adds latency due to sequential LLM calls

    Future work:
        - LLM-based routing between text-only and multimodal retrieval
        - Fine-tuned embeddings trained on MMDocIR domain data
        - Full test split evaluation for final reported numbers
        - Reranker layer (e.g. cross-encoder) between retrieval and generation

APPENDIX D — REPRODUCTION INSTRUCTIONS

    Environment setup:
        pip install -r requirements.txt

    Run the baseline system:
        python src/main_baseline.py

    Run the advanced system with a specific query technique:
        python src/main_advanced.py --technique standard
        python src/main_advanced.py --technique hyde
        python src/main_advanced.py --technique rag_fusion
        python src/main_advanced.py --technique multi_query
        python src/main_advanced.py --technique step_back
        python src/main_advanced.py --technique query_decomposition
        python src/main_advanced.py --technique query_rewriting
        python src/main_advanced.py --technique query_expansion

    To rebuild the vector index from scratch:
        Set force_rebuild = True in main_baseline.py or main_advanced.py
        before running.

    Vector database: stored locally in local_qdrant/
    Pre-processed chunks: stored in src/data/preprocessed/
